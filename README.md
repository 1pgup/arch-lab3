# Сервис синхронизации и отчётов по курсам валют ČNB

Приложение решает следующие задачи:

1. автоматически загружает курсы валют из ČNB по расписанию;
2. позволяет вручную загрузить данные за период;
3. сохраняет данные в SQLite;
4. отдаёт JSON-отчёт с минимальным, максимальным и средним значением курса для 1 условной единицы валюты.

## Используемые технологии

- Python 3.12
- FastAPI
- APScheduler
- SQLite
- Docker / docker-compose

## Структура проекта

```text
.
├── app/
│   ├── cnb_client.py      # загрузка и разбор файлов ČNB
│   ├── config.py          # чтение YAML-конфигурации
│   ├── config.yaml        # список валют и расписание
│   ├── database.py        # работа с SQLite
│   ├── main.py            # точки входа FastAPI
│   ├── scheduler.py       # запуск фонового планировщика
│   ├── schemas.py         # схемы запросов
│   └── services.py        # бизнес-логика синхронизации и отчётов
├── data/                  # здесь будет лежать SQLite база
├── docker-compose.yml
├── Dockerfile
└── requirements.txt
```

## Запуск

```bash
docker-compose up --build
```

После запуска сервис будет доступен по адресу:

- API: `http://localhost:8000`
- Swagger UI: `http://localhost:8000/docs`

## Конфигурация

Файл `app/config.yaml` содержит:

- путь к базе данных;
- список валют для синхронизации;
- режим расписания:
  - `cron` — запуск по времени;
  - `interval` — запуск через интервал в секундах.

Пример:

```yaml
database:
  path: "./data/exchange_rates.db"

sync:
  currencies:
    - USD
    - EUR
    - PLN

  scheduler:
    mode: "cron"
    hour: 0
    minute: 1
```

## API

### 1. Проверка состояния

`GET /health`

Пример ответа:

```json
{
  "status": "ok",
  "scheduler_mode": "cron",
  "currencies": ["USD", "EUR", "PLN", "GBP"]
}
```

### 2. Ручная синхронизация за один день

`POST /sync/daily`

Тело запроса:

```json
{
  "date": "2025-01-10"
}
```

Если `date` не передана, будет использована текущая дата.

### 3. Ручная синхронизация за период

`POST /sync/range`

Тело запроса:

```json
{
  "startDate": "2025-01-01",
  "endDate": "2025-01-31"
}
```

### 4. Отчёт по валютам

`GET /report?startDate=2025-01-01&endDate=2025-01-31&currencies=USD,EUR,PLN`

Пример ответа:

```json
{
  "period": {
    "startDate": "2025-01-01",
    "endDate": "2025-01-31"
  },
  "currencies": ["USD", "EUR", "PLN"],
  "results": {
    "USD": {
      "min": 24.11,
      "max": 24.83,
      "avg": 24.45,
      "samples": 21,
      "first_available_date": "2025-01-02",
      "last_available_date": "2025-01-31"
    }
  },
  "warnings": []
}
```

## Обработка пропусков и аномалий

В проекте предусмотрена мягкая обработка проблемных данных:

- пустые значения в исходном файле пропускаются;
- неполные строки не ломают синхронизацию;
- если за дату не найден `daily.txt`, приложение пробует несколько предыдущих дней;
- если в БД нет данных по запрошенной валюте, в отчёте возвращается предупреждение.
